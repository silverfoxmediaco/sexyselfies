Core Spacing System (Mobile-First)
The 4/8-Point Grid System
Use a 4- or 8-point grid system where spacing between elements (margins, paddings, gutters) should be multiples of 4 or 8 pixels (e.g., 4px, 8px, 16px, 24px). This ensures consistent spacing across different screen sizes.
Margin Guidelines by Viewport
* Mobile (< 768px): Margins typically range from 16px to 24px
* Tablet (768px - 1023px): 24px to 32px margins
* Desktop (≥ 1024px): Margins typically range from 30px to 60px on desktop
Padding Best Practices
* Interactive Elements: For buttons and interactive elements, padding of 8px to 16px on all sides ensures that elements are easily tappable on mobile devices
* Content Sections: Main content padding of 20px on mobile, 40px on tablets/desktop
* Cards/Containers: 16px to 24px internal padding
🎯 Touch Target Sizes (Critical for PWAs)
Minimum Touch Target Requirements
* WCAG Level AA: The size of the target for pointer inputs is at least 44 by 44 CSS pixels
* Google Material Design: Google's Material Design recognizes a minimum height of 48 Density-independent Pixels (dp)
* Best Practice: A minimum recommended touch target size is around 48 device independent pixels with 8 pixels spacing between targets
Position-Based Touch Targets
We need to aim for 11mm (or 31pt / 42px) on the top of the screen, and 12mm (or 34pt / 46px) at the bottom of the screen because users are least precise at screen edges.
Recommended Sizes by Position:
* Content Area Links: 27×27px minimum (30×30px preferred)
* Top Navigation: 44×44px minimum (48×48px preferred)
* Bottom Navigation: 46×46px minimum (48×48px preferred)
📐 Container Widths & Breakpoints
Standard Breakpoints (2024)
Key breakpoints: Mobile (320-767px), Tablet (768-1023px), Desktop (1024px+)
Container Max-Widths
Standard container max-widths:
* Small (sm): 540px
* Medium (md): 720px
* Large (lg): 960px
* Extra Large (xl): 1140px
* XXL: 1320px-1440px
PWA-Specific Recommendations:
/* Mobile First Container */
.container {
  width: 100%;
  padding: 0 16px; /* Mobile */
  margin: 0 auto;
}


@media (min-width: 768px) {
  .container {
    padding: 0 24px; /* Tablet */
    max-width: 720px;
  }
}


@media (min-width: 1024px) {
  .container {
    padding: 0 32px; /* Desktop */
    max-width: 1140px;
  }
}


@media (min-width: 1440px) {
  .container {
    max-width: 1320px; /* Large Desktop */
  }
}


🎨 Grid Systems & Gutters
Column Structure
* Mobile: 4- or 8-column grid for mobile designs
* Desktop: 12-column grid for web design
Gutter Spacing
Gutters (gaps between columns) are fixed values (8px, 16px or bigger) based on breakpoints
* Mobile: 8px to 16px gutters
* Tablet: 16px to 20px gutters
* Desktop: 20px to 30px gutters
📦 Card & Button Container Alignment
Card Layouts
/* Mobile-first card grid */
.card-grid {
  display: grid;
  gap: 16px; /* Mobile gap */
  grid-template-columns: 1fr;
  padding: 16px;
}


@media (min-width: 768px) {
  .card-grid {
    gap: 20px;
    grid-template-columns: repeat(2, 1fr);
    padding: 24px;
  }
}


@media (min-width: 1024px) {
  .card-grid {
    gap: 24px;
    grid-template-columns: repeat(3, 1fr);
    padding: 32px;
  }
}


Button Groups
.button-group {
  display: flex;
  gap: 12px; /* Consistent spacing */
  flex-wrap: wrap; /* Allow wrapping on mobile */
}


.button {
  min-height: 48px; /* Touch-friendly */
  padding: 12px 24px;
  flex: 1; /* Equal width on mobile */
  min-width: 120px;
}


@media (min-width: 768px) {
  .button {
    flex: initial; /* Natural width on larger screens */
  }
}


🎯 PWA-Specific Considerations
Safe Areas (for PWAs on devices with notches)
When you apply viewport-fit=cover, remember to use safe margins or paddings for critical content and interactive elements
.pwa-container {
  padding-top: env(safe-area-inset-top, 20px);
  padding-bottom: env(safe-area-inset-bottom, 20px);
  padding-left: env(safe-area-inset-left, 16px);
  padding-right: env(safe-area-inset-right, 16px);
}


Section Heights
* Hero Sections: 60-80vh on mobile, 70-90vh on desktop
* Content Sections: Min-height of 50vh, natural height based on content
* Footer: 200-300px on mobile, 300-400px on desktop
